## ADDED Requirements

### Requirement: 프로필 편집에 포트폴리오 섹션

프로필 편집 페이지(`/profile/edit`) 하단에 포트폴리오 관리 섹션을 추가해야 한다. 이 섹션은:

- 현재 사용자의 포트폴리오 항목 목록을 표시
- 최대 5개 항목까지 추가 가능
- 각 항목: 이미지(필수) + 제목(필수) + 설명 + 프로젝트 URL + 기술 태그
- 항목 추가/편집/삭제 기능

#### Scenario: 포트폴리오 항목 추가

- **WHEN** 사용자가 '포트폴리오 추가' 버튼을 클릭하고 이미지와 제목을 입력함
- **THEN** DB portfolios에 새 항목이 생성되고 imageUrl에 UploadThing URL이 저장됨

#### Scenario: 최대 5개 제한

- **WHEN** 사용자가 이미 5개의 포트폴리오를 가지고 있음
- **THEN** '포트폴리오 추가' 버튼이 비활성화되고 최대 개수 안내 메시지가 표시됨

#### Scenario: 포트폴리오 항목 편집

- **WHEN** 사용자가 기존 포트폴리오 항목의 편집 버튼을 클릭함
- **THEN** 인라인 편집 폼이 열리고 이미지 교체 포함 모든 필드를 수정할 수 있음

#### Scenario: 포트폴리오 항목 삭제

- **WHEN** 사용자가 포트폴리오 항목의 삭제 버튼을 클릭함
- **THEN** 확인 후 DB에서 항목이 삭제됨

### Requirement: 포트폴리오 이미지 업로드

각 포트폴리오 항목에 ImageUploader 컴포넌트를 사용하여 이미지를 업로드한다. 파일 크기 제한 4MB, 형식 JPG/PNG/WebP.

#### Scenario: 포트폴리오 이미지 업로드 성공

- **WHEN** 사용자가 포트폴리오에 이미지를 업로드함
- **THEN** UploadThing에 업로드되고 미리보기가 표시됨

#### Scenario: 포트폴리오 이미지 교체

- **WHEN** 기존 이미지가 있는 포트폴리오에서 새 이미지를 업로드함
- **THEN** 기존 이미지가 새 이미지로 교체됨

### Requirement: 작업자 상세에 포트폴리오 표시

작업자 상세 페이지(`/workers/:workerId`)에서 해당 작업자의 포트폴리오 항목을 그리드 형식으로 표시해야 한다. 각 항목은 썸네일 이미지 + 제목 + 설명을 보여준다.

#### Scenario: 포트폴리오가 있는 작업자

- **WHEN** 포트폴리오 항목이 있는 작업자의 상세 페이지를 봄
- **THEN** 포트폴리오 섹션에 이미지 그리드가 표시됨

#### Scenario: 포트폴리오가 없는 작업자

- **WHEN** 포트폴리오 항목이 없는 작업자의 상세 페이지를 봄
- **THEN** 포트폴리오 섹션이 표시되지 않거나 '아직 포트폴리오가 없습니다' 메시지가 표시됨
